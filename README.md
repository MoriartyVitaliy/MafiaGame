# МАФИЯ — онлайн игра

Веб-игра «Мафия» в стиле нуарного детективного досье. Играть можно с друзьями по сети — каждый открывает игру в браузере и заходит по коду комнаты.

## Как запустить локально

Нужен [Node.js](https://nodejs.org/) версии 18+.

```bash
npm install
npm start
```

Откройте `http://localhost:3000` в браузере. Чтобы поиграть с друзьями в одной локальной сети (Wi-Fi), они могут зайти по вашему локальному IP, например `http://192.168.1.23:3000`.

## Как играть по интернету (не в одной сети)

Сервер должен быть доступен из интернета. Три простых варианта:

### Вариант 1: ngrok (быстрее всего, для разовой игры)
```bash
npm start
# в другом терминале:
npx ngrok http 3000
```
Ngrok выдаст публичную ссылку вида `https://xxxx.ngrok-free.app` — отправьте её друзьям.

### Вариант 2: бесплатный деплой (Render.com)
1. Загрузите этот проект на GitHub.
2. На [render.com](https://render.com) создайте новый **Web Service** из репозитория.
3. Build command: `npm install`, Start command: `npm start`.
4. После деплоя Render даст постоянную публичную ссылку.

### Вариант 3: Railway / Fly.io
Аналогично Render — это платформы, которые бесплатно (в рамках лимитов) держат Node.js-сервер онлайн 24/7, что удобно, если планируете играть регулярно.

## Правила игры

- Минимум 4 игрока.
- Роли распределяются случайно: **Мафия**, **Доктор**, **Детектив**, **Мирные жители** (количество особых ролей растёт с числом игроков).
- **Ночь**: мафия выбирает жертву, доктор может спасти одного человека, детектив проверяет одного подозреваемого.
- **День**: обсуждение в чате.
- **Голосование**: город изгоняет одного подозреваемого.
- Мирные жители победят, если вся мафия будет устранена. Мафия победит, если её численность станет равна или больше числа мирных жителей.

## Структура проекта

```
server.js         — вся игровая логика и WebSocket-события (Socket.io)
public/index.html — разметка трёх экранов: лобби, комната ожидания, игра
public/style.css  — нуарный дизайн (стиль досье, штампы, эффект прослушки)
public/client.js  — вся клиентская логика и обработка событий сокета
```

## Возможные доработки

- Голосовой/видеочат (можно интегрировать WebRTC или встроить Jitsi).
- Роли Маньяк, Мэр, Проститутка и др.
- Персистентность через базу данных (сейчас всё хранится в памяти сервера — при перезапуске все комнаты сбрасываются).
- Ограничение по времени на реплику в чате, антифлуд.
